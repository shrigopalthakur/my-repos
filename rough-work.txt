AWS par do websites host karne ka process step-by-step samjhata hoon, jaise ki aapne kaha "noverse" aur "just press" ko host karna hai. Hum EC2 instance, Apache/Nginx web server, aur file upload process cover karenge. Aapko har step ka detail milega aur kis directory mein files upload karni hain, uske baare mein bhi.

Step 1: AWS Account Create Karna
Sign Up: AWS ki website par jaake apna account create karein. Agar aapke paas AWS account nahi hai toh Sign Up option pe click karein aur apni details (email, password, aur payment details) bhar ke account create karein.
Payment Info: AWS free tier mein kaafi resources milte hain (limited usage), lekin aapko payment info deni hogi.
Step 2: EC2 Instance Launch Karna
EC2 (Elastic Compute Cloud) ek virtual server hai jahan par aap apni websites host kar sakte hain.

AWS Management Console mein login karein.
Services menu mein se EC2 choose karein.
Launch Instance pe click karein.
Amazon Machine Image (AMI):
Aapko Ubuntu choose karna hoga, jise aapke website ke liye web server setup karne mein madad milegi.
Instance Type:
t2.micro (Free Tier eligible) select karein.
Configure Instance: Default settings ko chhod dijiye.
Add Storage: Default storage size (8 GB) sufficient hai.
Configure Security Group:
HTTP (port 80) aur SSH (port 22) open karein.
Agar aap HTTPS use karna chahte hain toh HTTPS (port 443) bhi open karein.
Launch: Key pair select karein (agar naya pair banana ho toh “Create a new key pair” option select karen aur download karen).
Step 3: EC2 Instance Ko Access Karna (SSH)
Apne EC2 instance ka public IP dhundhen.
SSH (Secure Shell) ke through apne instance se connect ho jaayein. Agar aap Windows use kar rahe hain toh PuTTY ka use karein, aur agar Mac ya Linux use kar rahe hain toh terminal ka use karein:
Example command (Mac/Linux):
bash
Copy
ssh -i /path/to/your-key.pem ubuntu@<Your-EC2-Public-IP>
Connect hone ke baad aapka terminal EC2 instance ke liye open ho jayega.
Step 4: Apache Web Server Install Karna
Update Packages:
bash
Copy
sudo apt update
Install Apache:
bash
Copy
sudo apt install apache2
Apache Service Start Karna:
bash
Copy
sudo systemctl start apache2
sudo systemctl enable apache2
Check Apache: Agar Apache sahi se install ho gaya hai, toh apne EC2 instance ka public IP address browser mein type karke Apache ka default page dekh sakte hain.
Step 5: Do Websites Host Karne Ke Liye Directories Banaana
Aapko do alag-alag websites ke liye alag-alag directories banani hogi.

Create Directories for Websites:

Noverse website ke liye directory:
bash
Copy
sudo mkdir /var/www/html/noverse
Just Press website ke liye directory:
bash
Copy
sudo mkdir /var/www/html/justpress
Set Permissions: Ensure karen ki Apache ko inn directories ko access karne ki permissions ho.

bash
Copy
sudo chown -R www-data:www-data /var/www/html/noverse
sudo chown -R www-data:www-data /var/www/html/justpress
Step 6: Website Files Upload Karna
Aapko apne website files (HTML, CSS, JavaScript, images, etc.) ko apni EC2 instance ki directories mein upload karna hoga.

Website Files (Example):
noverse ke liye: /var/www/html/noverse
justpress ke liye: /var/www/html/justpress
Aapko apne website ke HTML files, CSS files, images, JavaScript, etc. upload karni hain:

SFTP ya SCP ka use karen apni local machine se files upload karne ke liye. Example command:
bash
Copy
scp -i /path/to/your-key.pem /path/to/your-website-files/* ubuntu@<Your-EC2-Public-IP>:/var/www/html/noverse
Same command Just Press ke liye use karen:
bash
Copy
scp -i /path/to/your-key.pem /path/to/your-website-files/* ubuntu@<Your-EC2-Public-IP>:/var/www/html/justpress
Step 7: Virtual Hosts Configure Karna (Apache)
Aapko Apache mein virtual hosts configure karne honge taaki dono websites alag-alag domains par run ho sakein.

Create Virtual Hosts File for Noverse:

Apache configuration file /etc/apache2/sites-available/noverse.conf create karein:
bash
Copy
sudo nano /etc/apache2/sites-available/noverse.conf
Noverse ke liye configuration add karein:
apache
Copy
<VirtualHost *:80>
    ServerAdmin webmaster@noverse.com
    ServerName noverse.com
    DocumentRoot /var/www/html/noverse
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
Create Virtual Hosts File for Just Press:

/etc/apache2/sites-available/justpress.conf create karein:
bash
Copy
sudo nano /etc/apache2/sites-available/justpress.conf
Just Press ke liye configuration add karein:
apache
Copy
<VirtualHost *:80>
    ServerAdmin webmaster@justpress.com
    ServerName justpress.com
    DocumentRoot /var/www/html/justpress
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
Enable Virtual Hosts:

Enable virtual hosts aur Apache ko restart karein:
bash
Copy
sudo a2ensite noverse.conf
sudo a2ensite justpress.conf
sudo systemctl restart apache2
Step 8: DNS Setup Karna (Domain Name)
Agar aap custom domain names (noverse.com aur justpress.com) use kar rahe hain, toh aapko Route 53 ya kisi bhi DNS provider par A records set karne honge jo aapke EC2 instance ke public IP address par point karein.

Step 9: SSL Setup Karna (Optional)
Agar aap apni websites par HTTPS chahte hain:

AWS ACM (AWS Certificate Manager) se SSL certificate le sakte hain.
Apache mein SSL configure karne ke liye mod_ssl module ko enable karen aur SSL certificates ko configure karen.
Step 10: Testing Websites
Apne domains ko browser mein open karke test karein. Agar sab kuch sahi hai, toh dono websites aapke EC2 server par run kar rahi hongi.

Is tarah se aap apne do websites "noverse" aur "just press" ko AWS EC2 instance par successfully host kar sakte hain.

Step-by-Step Guide to Create CNAME Record in AWS Route 53:
Step 1: AWS Console mein Login Karein
AWS Console me login karein: AWS Management Console

AWS Console me Route 53 ko search karein aur open karein.

Step 2: Hosted Zone Select Karein
Route 53 dashboard me, "Hosted Zones" section me jayein.

Aapko apne domain medflick.com ka hosted zone yahan dikhai dega. Agar medflick.com ka hosted zone already Route 53 mein configured hai, to uspe click karein.

Important: Agar aapka domain GoDaddy ya kisi aur registrar par registered hai, to aapko pehle name servers update karne honge (ye step neeche bataya gaya hai).

Step 3: CNAME Record Create Karein
Hosted Zone me "Create record" par click karein.

Record details fill karein:

Record name: hmn (Yeh subdomain ka naam hai, isse hmn.medflick.com banega).
Record type: CNAME ko select karein.
Value: sis.tdn.gtranslate.net (Yeh target domain hai jahan aapka CNAME point karega).
TTL: Default TTL (300 seconds) chhodein.
Routing policy: Simple routing ko select karein.
Evaluate target health: No set karein.
"Create records" button par click karein.

Step 5: Update Name Servers on GoDaddy (Only if Domain is Registered with GoDaddy)
Agar aapka domain GoDaddy par registered hai, to aapko apne domain ke name servers ko AWS Route 53 ke name servers se update karna hoga.

GoDaddy account me login karein.
My Products section me apne domain ko select karein.
DNS Settings ya Manage DNS par click karein.
Nameservers section me, Custom Nameservers select karein aur Route 53 ke name servers daalein. AWS Console me Hosted Zone page pe aapko ye name servers milenge (jaise ns-1234.awsdns-56.org).
Changes save karne ke baad, DNS propagation mein time lag sakta hai (typically 1-24 hours).
Step 6: Verify CNAME Record
Jab DNS propagation complete ho jayegi, aap apne subdomain hmn.medflick.com ko sis.tdn.gtranslate.net ke saath point karte dekh sakenge.
Summary of Steps:
AWS Route 53 me Hosted Zone select karen.
CNAME record ko create karen: hmn.medflick.com → sis.tdn.gtranslate.net.
Agar aapka domain GoDaddy par registered hai, to name servers update karen.
DNS Checker se record ki propagation verify karen.
Wait for DNS Propagation (1-24 hours).

Command line method se cname add karna :-

Haan, aap AWS CLI ka use karke Route 53 me CNAME record add kar sakte hain, jo aapke Amazon Web Services (AWS) account ka part hai.

Agar aap AWS CLI ke through CNAME record add karna chahte hain, to aapko Route 53 ka use karna hoga, jo AWS ka managed DNS service hai.

Prerequisites:
AWS CLI ko apne machine par install karna hoga.

AWS account me Route 53 setup hona chahiye, aur aapke paas domain (yaha pe medflick.com) registered hona chahiye.

AWS CLI me apna account configure hona chahiye. Agar aapne pehle se AWS CLI configure nahi kiya hai, to command chalayein:

bash
Copy
aws configure
Yahan aapko AWS Access Key, Secret Key, Region, aur Output format enter karna hoga.

Step 1: CNAME Record Ka JSON File Prepare Karein
AWS CLI ke through CNAME record add karne ke liye aapko ek JSON file create karni hogi jo DNS record details ko define kare.

Example JSON File (cname-record.json):
json
Copy
{
  "Comment": "Add CNAME record for hmn.medflick.com",
  "Changes": [
    {
      "Action": "UPSERT",
      "ResourceRecordSet": {
        "Name": "hmn.medflick.com.",
        "Type": "CNAME",
        "TTL": 60,
        "ResourceRecords": [
          {
            "Value": "sis.tdn.gtranslate.net"
          }
        ]
      }
    }
  ]
}
Explanation:

"Name": "hmn.medflick.com." — Yahaan pe aapke subdomain ka naam specify kiya gaya hai.
"Value": "sis.tdn.gtranslate.net" — Yeh woh target value hai jahan aapka CNAME record point karega.
"TTL": 60 — Yeh Time To Live hai, jo record ke cache time ko define karta hai (in seconds).
"Action": "UPSERT" — Agar record already exist karta hai to use update karega, agar nahi to naya record create karega.
Step 2: CNAME Record AWS CLI Se Add Karein
Aapko Route 53 me apne hosted zone ID ki zarurat padegi. Hosted zone ID ko aap AWS Console ya CLI se fetch kar sakte hain.

Fetch Hosted Zone ID:
Agar aapke paas hosted zone ID nahi hai, to aap following command se apne hosted zones ko list kar sakte hain:

bash
Copy
aws route53 list-hosted-zones
Is command se aapko aapke domain ke liye hosted zone ID mil jayegi.

Add CNAME Record:
Ab, aap change-resource-record-sets command ko use karke CNAME record add kar sakte hain:

bash
Copy
aws route53 change-resource-record-sets \
    --hosted-zone-id Z3M3KABCDEF --change-batch file://cname-record.json
Yahan:

Z3M3KABCDEF — Aapka actual hosted zone ID hoga.
file://cname-record.json — Aapne jo JSON file prepare ki thi, uska path hai.
